# Purpose

This directory contains technical documentation related to the shared platform layer.

Its role is to explain how the platform is integrated, extended, and reused across projects.

# What goes here

Files in this directory may include:

- platform usage guides
- integration notes
- extension rules
- technical design notes for shared infrastructure

Typical examples:

- `platform-usage.md`

# What NOT goes here

This directory should NOT contain:

- product requirements
- business specifications
- user documentation
- project-specific architecture docs
- model documentation tied to one application

# Documentation

## Purpose
This folder contains project documentation that explains how the system is designed, why decisions were made, and how the main flows work.

## Main Areas
- `architecture/` - System structure and component relationships
- `flows/` - End-to-end flow explanations
- `decisions/` - Architecture and implementation decisions
- `diagrams/` - Visual assets and supporting diagrams

## Notes
Documentation should evolve with the system and stay aligned with actual implementation choices.

# Tests

## Purpose
This folder contains all testing-related assets for the project.

The goal is to validate both classic software behavior and RAG-specific quality concerns.

## Main Areas
- `unit/` - Small isolated tests for individual components
- `integration/` - Tests across connected components and flows
- `evaluation/` - RAG-specific quality checks and benchmark-style validation

## Notes
Testing in RAG systems should cover both correctness and answer quality, not only code execution.

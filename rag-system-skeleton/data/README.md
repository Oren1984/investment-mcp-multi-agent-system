# Data

## Purpose
This folder contains the data used by the RAG system across different processing stages.

It helps separate source materials from processed artifacts and example assets.

## Main Areas
- `raw/` - Original input documents and source files
- `processed/` - Cleaned, transformed, or chunk-ready outputs
- `samples/` - Small example data for demos, testing, or experiments

## Notes
Avoid mixing raw source files with transformed outputs.  
Clear separation makes debugging, reproducibility, and pipeline validation easier.
